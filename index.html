Iniciando com HTML

Concluí o módulo "Trabalhando com Formulários" do Bootcamp Santander 2025, voltado para o desenvolvimento web. Foi uma experiência breve, mas extremamente rica!

Principais aprendizados:

  • A estrutura básica de um formulário HTML (<form>, <input>, <label>, <textarea>, <select>, etc.).
  • A importância do atributo name para enviar dados corretamente Diferença entre os tipos de input (como text, radio, checkbox, email, password, entre outros).
  • Como aplicar validações simples diretamente no HTML com os atributos required, minlength, maxlength, pattern.
  • Uso do atributo action para definir o destino dos dados e method para especificar o tipo de requisição (GET ou POST).
  • Introdução ao uso do evento onsubmit para validar dados com JavaScript.

Esses conceitos são a base para desenvolver páginas interativas e funcionais, fundamentais para qualquer aplicação web.

Entendendo o HTML semântico

Durante esta etapa do bootcamp, aprofundei o uso de HTML semântico e tive uma introdução prática ao processo de Web Scraping.

Principais aprendizados:

  • Uso adequado de tags como <article>, <section>, <figure>, <figcaption>, <time> e <blockquote> para melhorar a estrutura semântica e a acessibilidade do conteúdo.
  • Aplicação do <picture> para imagens responsivas, adaptando o layout a diferentes tamanhos de tela.

Fundamentos do Web Scraping, incluindo:

  • Identificação do conteúdo-alvo em páginas web;
  • Análise do código-fonte (HTML);
  • Extração de dados com bibliotecas como BeautifulSoup, Scrapy e Selenium;
  • Organização dos dados extraídos em formatos estruturados (JSON, XML, Excel).

Esse conhecimento reforça a importância de estruturar páginas web de forma clara, acessível e funcional, além de abrir portas para aplicações mais avançadas com automação e análise de dados.

Acessibilidade em HTML: inclusão desde o código

Voltar

A acessibilidade na web garante que todas as pessoas, inclusive com deficiências visuais, auditivas, motoras ou cognitivas, possam navegar e interagir com páginas da internet de forma autônoma. No desenvolvimento HTML, isso envolve:

  • Uso correto de tags semânticas (<main>, <header>, <nav>, <article>, <section>, etc.), para que leitores de tela consigam entender a estrutura da página;
  • Adição de textos alternativos em imagens com alt="" para descrever o conteúdo visual;
  • Criação de formulários com <label> associado a <input>, permitindo que o usuário saiba o que preencher;
  • Contraste de cores adequado e navegação por teclado (tabulação acessível)

E o que é ARIA?

ARIA é um conjunto de atributos HTML que fornece informações adicionais de acessibilidade, especialmente útil quando elementos interativos não são naturalmente compreendidos por leitores de tela.

  • role="button" – informa que um <div> ou outro elemento age como botão.
  • aria-label="Fechar menu" – fornece uma descrição acessível mesmo sem texto visível.
  • aria-hidden="true" – esconde um conteúdo visual de tecnologias assistivas.
  • aria-expanded="false" – indica se um menu expansível está aberto ou fechado.

Boas práticas com ARIA:

  • Use primeiro HTML semântico. ARIA não substitui tags corretas — apenas complementa.
  • Evite usar ARIA sem necessidade, pois pode confundir leitores de tela se aplicado incorretamente.
  • Sempre teste a acessibilidade com ferramentas como o Lighthouse, NVDA ou o leitor de tela do seu sistema.
